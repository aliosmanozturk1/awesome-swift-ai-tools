# Contributing

Thanks for your interest in contributing! This is a curated list of AI tools, MCPs, agents, skills, rules, and prompts for Swift development., libraries, and resources.

## Ways to contribute
- Add a new project/resource link
- Fix broken links
- Improve descriptions or categorization
- Suggest new categories
- Report issues (duplicates, out-of-scope items, abandoned projects)

## Submission rules (quality bar)
A submission should:
- Be relevant to **Swift development** using **AI tools** (e.g., MCP servers, agents, skills, rules, prompts, IDE assistants like Claude Code/Cursor) for iOS/macOS projects.
- Be publicly accessible (no private/internal links).
- Have a clear description/README and a working link.
- Be reasonably maintained (not obviously abandoned).
- Not be spam, affiliate marketing, or link farms.

We may remove entries that become outdated, unmaintained, broken, or out of scope.

## Format
Please use exactly this format:

`- [Project Name](https://link) — One sentence description.`

Guidelines:
- **One line per item**.
- Start the description with a capital letter and **end with a period**.
- Keep it short (aim for < 120 characters).
- No “best / ultimate / revolutionary” marketing language.
- Use consistent naming (official project name).
- Prefer the canonical project page (GitHub repo, official docs, paper page, etc.).
- No tracking/shortened URLs.

## Ordering
- If a section is **alphabetical**, keep it alphabetical.
- Otherwise, add the new item to the **end of the most relevant section**.

## Pull requests
- **One item per PR** (unless you’re fixing multiple broken links or doing a focused cleanup).
- Title examples:
  - `Add <Project>`
  - `Fix broken link for <Project>`
  - `Reorganize <Section>`

### PR checklist
- [ ] Link works and points to the correct resource.
- [ ] Item matches the scope (Swift development + AI tools like MCPs/agents/skills/rules/prompts)
- [ ] Follows the exact format: `- [Name](url) — Description.`
- [ ] Description is one sentence and ends with a period.
- [ ] Added to the correct section and follows ordering rules.
- [ ] No duplicates.

## Reporting broken links / outdated entries
If you find a broken or outdated entry:
- Open an issue with the section name and the link, **or**
- Submit a PR removing/replacing it.

## Code of Conduct
By participating, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
